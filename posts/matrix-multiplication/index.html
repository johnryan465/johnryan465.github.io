The Complexity of Matrix Multiplication

📅Tue Oct 13 2020

Linear Algebra
Maths

Matrix Multiplication is something which you may never really have considered too much, it is how we calculate the composition of 2 linear transformations. The algorithm is pretty simple and doesn't seem inefficient in any real way. However there is quite a bit more than meets the eye to this.

Definition

We will be considering the following linear maps

LA:W→U,LB:V→W,LC:V→UL_A : W \rightarrow U, L_B : V \rightarrow W, L_C : V \rightarrow ULA​:W→U,LB​:V→W,LC​:V→U

n=dim(W),m=Dim(V),p=dim⁡(U)n = dim(W), m = Dim(V), p= \dim(U)n=dim(W),m=Dim(V),p=dim(U)

We want to find LC=LA∘LBL_C = L_A \circ L_BLC​=LA​∘LB​, in particular we actually want to compute the matrix CCC that represents LCL_CLC​ from the matrices A,BA,BA,B.

Starting point

The method for matrix multiplication we all know is as follows.

cij=∑k=1naikbkjc_{ij} = \sum_{k=1}^n a_{ik}b_{kj}cij​=∑k=1n​aik​bkj​

To compute each entry of CCC, we perform nnn multiplications and n−1n-1n−1 additions. We must do this for all m×pm \times pm×p entries of CCC. So we have npmnpmnpm multiplications and (n−1)mp(n-1)mp(n−1)mp additions.

This leads to an algorithm with computational complexity of O(n3)O(n^3)O(n3) for multiplication of 2 n×nn \times nn×n matrices.

There doesn't seem to really be any redundant calculations, but actually improvements on this exist.

Stratten's Algorithm

Strattens Algorithm was the first sub-cubic complexity algorithm for matrix multiplication. We can write matrix multiplication as 888 sub-matrix multiplications and 444 matrix additions. This algorithm is also O(n3)O(n^3)O(n3), we can quite easily prove this.

[C1,1C1,2 C2,1C2,2]=[A1,1A1,2 A2,1A2,2][B1,1B1,2 B2,1B2,2] =[A1,1B1,1+A1,2B2,1A1,1B1,2+A1,2B2,2 A2,1B1,1+A2,2B2,1A2,1B1,1+A2,2B2,2]\begin{aligned}\begin{bmatrix} C_{1,1} & C_{1,2} \\\ C_{2,1} & C_{2,2} \end{bmatrix} &= \begin{bmatrix} A_{1,1} & A_{1,2} \\\ A_{2,1} & A_{2,2} \end{bmatrix} \begin{bmatrix} B_{1,1} & B_{1,2} \\\ B_{2,1} & B_{2,2} \end{bmatrix} \\\ &= \begin{bmatrix} A_{1,1}B_{1,1} + A_{1,2}B_{2,1} & A_{1,1}B_{1,2} + A_{1,2}B_{2,2} \\\ A_{2,1}B_{1,1} + A_{2,2}B_{2,1} & A_{2,1}B_{1,1} + A_{2,2}B_{2,2} \end{bmatrix} \end{aligned}[C1,1​ C2,1​​C1,2​C2,2​​] ​=[A1,1​ A2,1​​A1,2​A2,2​​][B1,1​ B2,1​​B1,2​B2,2​​]=[A1,1​B1,1​+A1,2​B2,1​ A2,1​B1,1​+A2,2​B2,1​​A1,1​B1,2​+A1,2​B2,2​A2,1​B1,1​+A2,2​B2,2​​]​

Strattens Algorithm works by instead of doing 888 sub matrix multiplications it does 777, this is in exchange for more additions.

The matrix multiplications are defined as follows.

M1=(A1,1+B2,2)(B1,1+B2,2) M2=(A2,1+A2,2)B1,1 M3=A1,1(B1,2−B2,2) M4=A2,2(B2,1−B2,2) M5=(A1,1+A1,2)B2,2 M6=(A2,1−A1,1)(B1,1+B1,2) M7=(A1,2−A2,2)(B2,1+B2,2)\begin{aligned}M_1 &= (A_{1,1} + B_{2,2})(B{1,1} + B_{2,2}) \\\ M_2 &= (A_{2,1} + A_{2,2})B_{1,1} \\\ M_3 &= A_{1,1}(B_{1,2} - B_{2,2}) \\\ M_4 &= A_{2,2} (B_{2,1} - B_{2,2}) \\\ M_5 &= (A_{1,1} + A_{1,2})B_{2,2} \\\ M_6 &= (A_{2,1} - A_{1,1})(B_{1,1} + B_{1,2}) \\\ M_7 &= (A_{1,2} - A_{2,2})(B_{2,1} + B_{2,2})\end{aligned}M1​ M2​ M3​ M4​ M5​ M6​ M7​​=(A1,1​+B2,2​)(B1,1+B2,2​)=(A2,1​+A2,2​)B1,1​=A1,1​(B1,2​−B2,2​)=A2,2​(B2,1​−B2,2​)=(A1,1​+A1,2​)B2,2​=(A2,1​−A1,1​)(B1,1​+B1,2​)=(A1,2​−A2,2​)(B2,1​+B2,2​)​

The we have

C1,1=M1+M4−M5+M7 C1,2=M3+M5 C2,1=M2+M4 C2,2=M1−M2+M3+M6\begin{aligned}C_{1,1} &= M_1 + M_4 - M_5 + M_7 \\\ C_{1,2} &= M_3 + M_5 \\\ C_{2,1} &= M_2 + M_4 \\\ C_{2,2} &= M_1 - M_2 + M_3 + M_6\end{aligned}C1,1​ C1,2​ C2,1​ C2,2​​=M1​+M4​−M5​+M7​=M3​+M5​=M2​+M4​=M1​−M2​+M3​+M6​​

We can verify that this holds by matrix algebra is quite easily, and them find that the complexity of the algorithm is O(nlog⁡2(7))O(n^{\log_2(7)})O(nlog2​(7)).

This is ... deeply unsatisifiying, atleast to me. The seemingly arbitrary matrix products don't really seem to give any insight as to what is occurring and why it is better than the naive approach.

A linear transformation is completely defined by its actions on a basis, in this case the basis is of size nnn. Computing each of the transformations on the basis seperately takes O(n2)O(n^2)O(n2) operations. So we are somehow taking advantage of some inherent structure in matrix multiplication to be able to calculate this using a sub-cubic algorithm.

We wish to determinine the smallest ω\omegaω such that matrix multiplication is O(nω)O(n^\omega)O(nω)

But what structure? Is there similarly sequence of 666 sub-matrix multiplications that we can use to create an even better matrix multiplication algorithm?

Tensors and Multi-linear Maps

Matrices represent linear maps from one vector space to another. Matrix Multiplication takes 2 linear maps, and maps this to another one, lets call this mapping MMMMMM.

More formally.

LA:W→U,LB:V→W,LC:V→UL_A : W \rightarrow U, L_B : V \rightarrow W, L_C : V \rightarrow ULA​:W→U,LB​:V→W,LC​:V→U

MM:(W→U)×(V→W)→(V→U)MM : (W \rightarrow U) \times (V \rightarrow W) \rightarrow ( V \rightarrow U)MM:(W→U)×(V→W)→(V→U)

Each of these linear maps are themselves elements of a vector space.

MMV,W,U:A×B→C,A∈L(W,U),B∈L(V,W),C∈L(V,U)MM_{V,W,U} : A \times B \rightarrow C, A \in L(W,U), B \in L(V,W), C \in L(V,U)MMV,W,U​:A×B→C,A∈L(W,U),B∈L(V,W),C∈L(V,U)

We have λC=(λA)B=A(λB),λ∈F\lambda C = (\lambda A)B = A(\lambda B), \lambda \in FλC=(λA)B=A(λB),λ∈F. This means that our mapping MMMMMM is bilinear in its 2 inputs.

Note: From here on out we will pretty much be be working with the case of multiplying 2 n*n matrices over the same field. In the spirit of this we can instead write our Tensor as ⟨n⟩\langle n \rangle⟨n⟩ if all dimensions are equal or ⟨m,n,p⟩\langle m,n,p \rangle⟨m,n,p⟩, where FFF is our field and m,n,pm,n,pm,n,p are the dimensions we are working over, we will call this space VVV

Tensors

A bit of an introduction to tensors for anyone who hasn't seen them before.

We define a tensor of TTT over vector spaces {V1,…,Vn}\{V_1,\ldots,V_n\}{V1​,…,Vn​} over a common field FFF as follows

T:V1×…×Vn→FT : V_1 \times \ldots \times V_n \rightarrow F T:V1​×…×Vn​→F

Note: more restrictive versions of this definition exist where the vector spaces used are restricted to multiple copies of a vector space and its dual

TTT is a multi-linear map, it is linear in each of its inputs.

By fixing a basis for {V1,…,Vn}\{V_1,\ldots,V_n\}{V1​,…,Vn​} we get a multidimensional array ddd of dimensions nnn, with the entries of this array being elements of FFF.

T=∑u=1dim⁡V1…∑w=1dim⁡Vndu…vxu⊗…⊗zwT = \sum_{u=1}^{\dim V_1} \ldots \sum_{w=1}^{\dim V_n} d_{u \ldots v} {x}_u \otimes \ldots \otimes z_wT=∑u=1dimV1​​…∑w=1dimVn​​du…v​xu​⊗…⊗zw​.

where {x,…,z}\{x, \ldots, z\}{x,…,z} are the corresponding basis for the vector spaces.

Using this form we can rewrite the matrix multiplication tensor as

⟨m,n,p⟩=∑i=1m∑j=1p∑k=1naik⊗bkj⊗cij\langle m,n,p \rangle = \sum_{i=1}^{m} \sum_{j=1}^{p} \sum_{k=1}^{n} a_{ik} \otimes b_{kj} \otimes c_{ij}⟨m,n,p⟩=∑i=1m​∑j=1p​∑k=1n​aik​⊗bkj​⊗cij​

We have now shown that we can talk about matrix multiplication as a multi-linear map (bi-linear) and in turn a tensor, which is cool I guess but seems like quite a tangent.

A theorem approaches

The tensor representation seems a bit of of left field, however by using this we can bring some powerful maths tools to bear. But before we can do this, we unfortunately need more definitions.

First we will need to define what the rank of a tensor is.

It is the minimum number of rank 1 tensors required to represent our tensor. rk(t)=minR.t=∑r=1Rλra1,r⊗…aM,rrk(t) = min_R.t = \sum^R_{r=1} \lambda_r a_{1,r} \otimes \ldots a_{M,r} rk(t)=minR​.t=∑r=1R​λr​a1,r​⊗…aM,r​

The border rank is then minimum rank of a sequence of tensors of which the limit is is the tensor in question.

Clearly we have brk(t)≤rk(t)brk(t) \leq rk(t)brk(t)≤rk(t), as the sequence which is just the tensor with its rank rrr decompisition repeated gives a feasible border rank of rrr

We also have the simple result that rk(t)≤∏i=1Mdim(Vi)rk(t) \leq \prod^M_{i=1} dim(V_i)rk(t)≤∏i=1M​dim(Vi​). We get this by creating a rank 1 tensor corresponding to the each of the basis vector of each constituent vector space.

Restrictions

Rank and border rank of tensors are unfortunately not (yet) as well understood as the rank of matrices. However even without complete results, we can make some progress.

Useful facts

  • The number of elementary multiplications required for a bilinear map (which matrix multiplication is), is tightly bounded by the rank of the corresponding tensor.
  • The rank of a tensor product is sub multiplicative.

The first of these is very useful, it shows that attempting to bound the rank of the matrix multiplication tensor is pretty much exactly what we want to do to determine ω\omegaω.

The sub-multiplicative result implies that if we can get a good bound on the rank of any ⟨n⟩\langle n \rangle⟨n⟩ for a specific nnn we can use this to bound ω\omegaω.

We can relate the a specific property of this tensor to our search for efficent ways of multiplying 2 matrices.

Strassans Theorem

ω=lim⁡n→∞log⁡n(brk(⟨n⟩)) \omega = \lim_{n \rightarrow \infty} \log_n (brk(\langle n \rangle))ω=limn→∞​logn​(brk(⟨n⟩))

Where brk(t)brk(t)brk(t)is the border rank of the tensor.

It has been shown that rk(⟨2⟩))=7rk(\langle 2 \rangle)) = 7rk(⟨2⟩))=7, this fact and the sub-multiplicity of tensor rank allows us to bound ω≤log⁡2(7)\omega \leq \log_2(7)ω≤log2​(7). This is the same bound we get by using Stattens Algorithm!

This is actually what Stratten's Algorithm is doing. We are decomposing the ⟨2⟩\langle 2 \rangle⟨2⟩ tensor into 777 rank 1 tensors. The seemingly random matrixes represent this decomposition!
